# HTTP 상태 코드, 제대로 쓰는 법
200만 쓰는 API는 절반은 거짓말을 하고 있다. | 2026-04-01

### 개념
HTTP 상태 코드는 서버가 클라이언트에게 보내는 신호다. 단순히 성공/실패가 아니라, 정확한 코드를 쓰면 프론트엔드 · 모니터링 · 디버깅까지 훨씬 쉬워진다.
백엔드 개발자라면 상황에 맞는 코드를 골라 쓰는 습관이 필요하다.

2xx - 성공
값 | 뜻 | 의미
---|:---:|---:
`200` | OK | 조회, 수정 성공. 가장 일반적인 성공.
`201` | Created | 리소스 생성 성공. POST로 새 데이터를 만들 떄.
`204` | No Content | 성공했지만 응답 바디 없음. 삭제할 때 딱!


3xx - 리다이렉트
값 | 뜻 | 의미
---|:---:|---:
`301` | Moved Permanently | URL이 영구적으로 바뀜. SEO에 영향.
`302` | Found | 임시 리다이렉트. 로그인 후 원래 페이지로.


4xx - 클라이언트 잘못
값 | 뜻 | 의미
---|:---:|---:
`400` | Bad Request | 요청 형식 자체가 잘못됨.
`401` | Unauthorized | 인증 안됨. 로그인이 필요.
`403` | Forbidden | 인증은 됐지만 권한 없음. (401과는 다름!)
`404` | Not Found | 리소스 없음.

5xx - 서버 잘못
값 | 뜻 | 의미
---|:---:|---:
`500` | Internal Server Error | 서버 내부 오류. 원인 모를 때 기본값.
`503` | Service Unavailable | 서버 과부화 및 점검 중.
