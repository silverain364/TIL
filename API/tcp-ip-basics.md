# TCP/IP - 인터넷이 작동하는 기본 원리
HTTP도, DB 연결도 결국 TCP에서 돌아간다! | 2026-04-07

### 개념
인터넷에서 데이터를 주고받는 건 편지를 보내는 것과 같다. IP는 주소(어디로 보낼지), TCP는 배송 방식(잘 도착했는지 확인)이다.
HTTP, HTTPS, DB 커넥션 모두 이 TCP/IP 위에서 동작한다.

### 계층 구조 - 4계층 모델
| 모델 | 종류 |
| --- | :---: |
| 응용 계층 | HTTP, HTTPS, FTP, DNS - 우리가 직접 쓰는 프로토콜
| 전송 계층 | TCP(신뢰성), UDP(속도) - 포트 번호로 프로세스 구분
| 인터넷 계층 | IP - 패킷을 목적지 IP 주소로 라우팅
| 네트워크 계층 | 이더넷, Wi-Fi - 실제 물리적 전송

### TCP 3-WAY HANDSHAKE - 연결이 맺어지는 과정
`SYN` → `SYN-ACK` → `ACK` = 연결 완료!
> 클라이언트가 "연결할게" (SYN) → 서버가 "좋아, 나도 연결할게" (SYN-ACK) → 클라이언트가 "확인" (ACK). HTTP 요청은 이 과정 이후에야 시작된다. HTTPS는 여기에 TLS 핸드셰이크가 추가된다.

### TCP VS UDP - 언제 뭘 쓰나
```
TCP - 순서 보장, 재전송 있음, 느림 : HTTP, HTTPS, DB 연결,파일 전송
UDP - 순서 무보장, 재전송 없음 빠름 : DNS 조회, 스트리밍, 게임, 영상통화
```
> 백엔드에서 포트가 안 열려 있을 떄 → netstat -tulpn | grep 8080으로 확인. TCP 연결이 안 되면 방화벽이나 포트 바인딩 문제 가능성
> 잘 알려진 포트 - HTTP:80, HTTPS:443, MySQL:3306, PostgreSQL:5432, Redis:6379, SSH:22
